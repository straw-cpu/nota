§1  理论 — GPC 模型基础Clarke et al. 1987
★ CARIMA 模型

GPC 建立在 CARIMA（Controlled Auto-Regressive Integrated Moving Average）模型上：

$$A(q^{-1})\,y(t) = B(q^{-1})\,u(t-k) + \frac{C(q^{-1})}{\Delta}\,\xi(t) \tag{1.1}$$

其中 $\Delta = 1-q^{-1}$，$C=1$（简化）。对 $\Delta$ 作用后等价于：

$$\tilde A(q^{-1})\,\Delta y(t) = B(q^{-1})\,\Delta u(t-k) + \xi(t) \tag{1.2}$$

na=1 时 $\tilde A$ 为二阶：$\tilde A = 1 + (a_1-1)q^{-1} - a_1 q^{-2}$，系数 $[1,\ -1.9846,\ 0.9846]$。

★ 丢番图递推

令 $1 = E_j\tilde A + q^{-j}F_j$，$G_j = E_j B$，则 $j$ 步预测：

$$\hat y(t+j|t) = G_j(q^{-1})\,\Delta u(t+j-k-1) + F_j(q^{-1})\,y(t) \tag{1.3}$$
推导：F/G 递推公式

初值 j=1：$E_1=1$，$F_1=[1-a_1,\ a_1]=[1.9846,\ -0.9846]$，$G_1=[b_0]$。

递推 j→j+1：令 $e = F_{j-1}^{(0)}$，

$$F_j^{(i)} = F_{j-1}^{(i+1)} - e(\tilde a_{i+1}-\tilde a_i), \quad G_j = [G_{j-1},\ e\cdot b_0]$$

$F_j$ 始终 2 项（na+1），$G_j$ 第 $j$ 步 $j$ 项。$\blacksquare$

★ 阶梯式 GPC 控制律（无矩阵求逆）

引入阶梯假设 $\Delta u(t+i)=\beta^i\delta$，柔化设定值 $W_j = \alpha^j y(t)+(1-\alpha^j)SP$：

$$\delta = \frac{\mathbf{g}_2^T(\mathbf{W}-\mathbf{Y}^1)}{\mathbf{g}_2^T\mathbf{g}_2 + \lambda\sum_{i=0}^{p_u-1}\beta^{2i}}, \quad \mathbf{g}_2 = \mathbf{G}_1\boldsymbol{\beta} \tag{1.4}$$
§2  实验适配 — 系统辨识数据：6_517_1.txt
★ ARX 模型辨识

信号归一化（先归一化再辨识，避免量纲影响）：

$$y_\text{norm} = \frac{y - y_\min}{y_\max - y_\min},\quad u_\text{norm} = \frac{u - u_\min}{u_\max - u_\min}$$

归一化范围：$y \in [1.0,\ 4.634]$ V，$u \in [4,\ 20]$ mA。ARX 模型结构（na=1, nb=0, k=24）：

$$y(t) = -a_1\,y(t-1) + b_0\,u(t-24)$$
辨识结果（归一化空间）
$a_1 = -0.984555$（极点 0.9846，时间常数 $\tau \approx 64$ 步）
$b_0 = 0.024361$，稳态增益 $K_{ss} = b_0/(1+a_1) = 1.577$
RMSE 验证：0.0164 V，时延 $k=24$ 步（24 s）
○ 阶次选择

AIC 最优阶次 na=4，但 na=1 与之 RMSE 差 <5%，且 GPC 推导最简，选 na=1。

符号约定（易错）
辨识出 $\theta_0 = |a_1| = 0.984555 > 0$，但多项式 $A = [1,\ -\theta_0]$，即 $a_1 < 0$。混淆符号是 §2 bug 的根源。

○ Bug 记录
Bug 1：F_j 维度错误
y 完全不跟踪 SP。根因：F_j 写成 na+2=3 项（多一项），递推越界。修复：F_j 长度严格为 na+1=2 项。
Bug 2：G_mat 索引偏移
Python 与 C++ 结果相差约 4 倍。根因：矩阵第 $j$ 行第 $i$ 列用了 $G_j[j-i+1]$（多移一位）。修复：改为 $G_j[j-i]$（0-indexed）。
Bug 3：RLS 参数漂移
$b_0$ 从 0.024 漂至 0.011，控制退化。根因：$\lambda_{RLS}=0.98$ 有效窗口仅 50 步。修复：改为 $\lambda_{RLS}=0.995$。
§4  仿真 — Python ARX 闭环验证gpc_na1.py
★ 主循环结构 
for t in range(N_sim):
    # ① RLS 参数 → A → 丢番图
    A_cur = [1.0, -rls.theta[0]]
    F_list, G_list = diophantine(A_cur, rls.theta[1], p)
    # ② 自由响应 Y¹
    Y1 = free_response(F_list, G_list, y_buf, du_buf, p, k)
    # ③ 柔化设定值
    W = soften_setpoint(y_buf[-1], SP, alpha, p)
    # ④ 阶梯式 GPC
    du_t = stepped_gpc(G_list, Y1, W, p, pu, lam, beta)
    du_t = clip(du_t, -DU_MAX, DU_MAX)
    u_t  = clip(u_prev + du_t, 0.0, 1.0)
    du_t = u_t - u_prev
    # ⑤ ARX 仿真一步
    y_t = -A_cur[1]*y_buf[-1] + rls.theta[1]*u_buf[-(k+1)]
    y_buf.append(y_t); du_buf.append(du_t)
    # ⑥ RLS 更新（控制律之后）
    rls.update([y_buf[-2], u_buf[-(k+1)]], y_t)
关键顺序
RLS 更新必须在控制律计算之后，否则用当前输出"预知"参数，产生超前偏差。

§5  工程实现 — C++ 嵌入式移植gpc_na1.cpp
★ RLS C++ 实现（2×2 手展开，零动态分配） 
static double P00=1e4,P01=0,P10=0,P11=1e4;
static double a1_hat=0.984555, b0_hat=0.024361;

void rls_update(double phi0, double phi1, double y) {
    double Pp0=P00*phi0+P01*phi1, Pp1=P10*phi0+P11*phi1;
    double denom=RLS_LAM+phi0*Pp0+phi1*Pp1;
    double K0=Pp0/denom, K1=Pp1/denom;
    double e=y-(a1_hat*phi0+b0_hat*phi1);
    a1_hat+=K0*e; b0_hat+=K1*e;
    P00=(P00-K0*Pp0)/RLS_LAM; P01=(P01-K0*Pp1)/RLS_LAM;
    P10=(P10-K1*Pp0)/RLS_LAM; P11=(P11-K1*Pp1)/RLS_LAM;
}
★ 调参速查
参数推荐值调节方向
$p$（预测时域）40≥ 2τ，一般不动
$\beta$（阶梯衰减）0.7振荡→减小，慢→增大
$\lambda$（控制权重）2振荡→增大，慢→减小
$\alpha$（柔化因子）0.9超调→增大，慢→减小
$\lambda_{RLS}$（遗忘因子）0.995漂移→增大至 0.999
$\Delta u_\max$（增量限幅）0.1冲击大→减小
推荐调参顺序
① 稳态确认（检查 $b_0$、归一化）→ ② 响应速度（$\lambda$↓ 或 $\beta$↓）→ ③ 压超调（$\alpha$↑）→ ④ 平滑控制量（$\Delta u_\max$↓）
